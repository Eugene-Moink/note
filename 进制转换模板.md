## 题型名称

- **进制字符串处理 / 数位处理（Base-N digit processing）**
- 常见任务：合法性判断、转十进制、展开式输出、比较大小、加减乘等（大数）

---

## 0. 先判断题目属于哪一类

1) **只是合法性判断**：每位数码是否 < base
2) **M 进制 → 十进制**：可能溢出吗？
3) **十进制 → N 进制**：数大不大？
4) **A 进制 → B 进制**：通常做法是先转“中间表示”（十进制大整数/任意进制大整数）

---

## 1) 必备工具函数：字符 ↔ 数码值

### char → digit（支持 0-9, A-Z, a-z）

```cpp
int val(char c){
    if('0'<=c && c<='9') return c-'0';
    if('A'<=c && c<='Z') return c-'A'+10;
    if('a'<=c && c<='z') return c-'a'+10;
    return -1;
}
```

### digit → char（常用到 36 进制）

```cpp
char dig(int d){
    if(d<10) return '0'+d;
    return 'A'+(d-10);
}
```

---

## 2) 模板 A：判断字符串是否可能是 base 进制

```cpp
bool okBase(const string& s, int base){
    for(char c: s){
        int d = val(c);
        if(d<0 || d>=base) return false;
    }
    return true;
}
```

---

## 3) 模板 B：base 进制字符串 → 十进制（能装进 long long 时）

**滚动累加：** `ans = ans*base + digit`

```cpp
long long toDecLL(const string& s, int base){
    long long ans = 0;
    for(char c: s){
        int d = val(c); // 默认你已保证合法
        ans = ans*base + d;
    }
    return ans;
}
```

注意：如果题目数据可能超 `long long`，就不能用这个，要用高精。

---

## 4) 模板 C：十进制 long long → base 进制字符串

**短除法：** 不断 `%base` 取最低位

```cpp
string fromDecLL(long long x, int base){
    if(x==0) return "0";
    string out;
    while(x>0){
        int r = x%base;
        out.push_back(dig(r));
        x/=base;
    }
    reverse(out.begin(), out.end());
    return out;
}
```

---

## 5) 模板 D：十进制“大整数字符串” → base 进制（高精）

适用：十进制输入位数很长。核心是“十进制字符串除以 base”。

**divmod：十进制字符串 ÷ base**

```cpp
pair<string,int> divmod10(const string& dec, int base){
    string q;
    long long cur=0;
    for(char c: dec){
        cur = cur*10 + (c-'0');
        int digit = cur/base;
        cur %= base;
        if(!q.empty() || digit!=0) q.push_back('0'+digit);
    }
    if(q.empty()) q="0";
    return {q,(int)cur};
}
```

**主过程：**

```cpp
string fromDecBig(string dec, int base){
    if(dec=="0") return "0";
    string out;
    while(dec!="0"){
        auto [q,r] = divmod10(dec, base);
        out.push_back(dig(r));
        dec=q;
    }
    reverse(out.begin(), out.end());
    return out;
}
```

---

## 6) A 进制 → B 进制的一般套路（最常见做法）

- 如果数不大：`A进制 -> long long十进制 -> B进制`
- 如果数很大：需要高精中间表示（通常用“大整数在 base B 或 base 10 的高精运算”），视题目而定。

---

## 7) 防坑清单（做任何进制题都先过一遍）

1) **输入位数很长**：用 `string`，别用整数读
2) **先合法性检查**：digit < base
3) **是否会溢出**：能否用 `long long`？不行就高精
4) **x=0 特判**（否则输出空串）
5) **字母大小写**：题目要 A-F 还是 a-f，输出一致
6) **前导 0**：合法，但转换输出通常要去掉（除非题目要求保留）
