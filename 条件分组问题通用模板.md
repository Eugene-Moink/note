## 📋 条件分组问题通用模板笔记

### 🔍 问题特征识别

当题目同时满足以下条件时，考虑**贪心+排序+双指针**策略：

- ✅ 需要**最大化分组数量**（或最小化未分组人数）
- ✅ 每组有**基于极值的条件**（如 `max ≥ k*min`）
- ✅ 每组人数固定或有上限（常见2人组）
- ✅ 数据范围较大（n ≤ 10^5，需O(n log n)解法）

### 🧠 核心思想

> **"让要求低的元素让步，把刚好够用的资源留给要求高的元素"**

**关键观察**：

1. **排序必要性**：消除顺序干扰，暴露数值关系
2. **贪心最优性**：小元素配对**最小的**满足条件的大元素，为后续留下更多选择
3. **双指针优势**：O(n)扫描完成配对，避免O(n²)暴力

### ⚙️ 通用算法步骤

```markdown
1. **预处理**：
   - 将数组按升序排序
   - 特殊处理：n=1时直接返回0（1人无法满足组条件）

2. **指针初始化**：
   - l = 0（指向当前待配对的最小元素）
   - r = 预处理位置（第一个可能满足条件的位置）
   - groups = 0（成功分组计数器）

3. **预处理右指针**：
   - 移动r直到满足组条件（或到达边界）
   - 目的：避免无效扫描，提高效率

4. **贪心配对**：
   while (l < n && r < n):
      if 当前配对满足条件:
          groups += 1
          l++, r++   // 两个元素都被使用
      else:
          r++        // 当前大元素不够，尝试更大的
      // 边界保护：确保 l < r

5. **输出结果**：
   - 直接输出groups
   - 注意：无法分组时groups=0（符合题意）
```

### 💻 代码模板（C++）

```cpp
#include<bits/stdc++.h>
#define endl "\n"
using ll = long long;
using namespace std;

int main()
{
    ll n;
    cin >> n;
    vector<ll> arr(n);
    for (int i = 0; i < n; i++)
    {
        cin >> arr[i];
    }

    // 边界情况：1人无法分组
    if (n < 2)
    {
        cout << 0 << endl;
        return 0;
    }

    // 1. 排序（核心基础）
    sort(arr.begin(), arr.end());
  
    // 2. 指针初始化
    ll l = 0;
    ll r = 1;
    ll groups = 0;
  
    // 3. 预处理：找到第一个满足条件的位置
    // 条件：arr[r] >= k * arr[l]（k根据题目变化）
    const ll k = 2;  // 本题要求 max >= 2*min
    while (r < n && arr[r] < k * arr[l])
    {
        r++;
    }
  
    // 4. 贪心配对核心
    while (l < n && r < n)
    {
        if (arr[r] >= k * arr[l])
        {
            groups++;  // 成功分组
            l++;       // 小元素已使用
            r++;       // 大元素已使用
        }
        else
        {
            r++;  // 当前大元素不够，寻找更大的
        }
      
        // 5. 边界保护：确保指针有效且l < r
        if (l >= r)
        {
            r = l + 1;
        }
    }
  
    // 6. 输出结果
    cout << groups << endl;
  
    return 0;
}
```

### 🔄 常见变体与调整

| 问题变体               | 修改点                 | 示例题目                       |
| ---------------------- | ---------------------- | ------------------------------ |
| **组条件变化**   | 修改k值或条件表达式    | max ≥ 3*min → k=3            |
| **每组人数>2**   | 需要多指针或队列       | 3人组：max ≥ 2*min + mid      |
| **最小化分组数** | 贪心策略反转           | 让大元素配最小满足条件的小元素 |
| **允许未分组**   | 不需要强制所有元素配对 | 输出最大可分组数               |
| **二维条件**     | 需要双重排序+双指针    | (a_i + b_i) ≥ k*(a_j + b_j)   |

### ⚠️ 重要注意事项

1. **排序方向**：

   - 升序排序 + 从小到大配对（本模板）
   - 降序排序 + 从大到小配对（某些变体更优）
2. **指针移动陷阱**：

   - 配对成功时**两个指针都要移动**
   - 避免 `l >= r`的无效状态（用边界保护）
3. **条件严格性**：

   - 注意题目是 `≥`还是 `>`（本题是 `≥`）
   - 浮点数问题：整数题通常无此问题
4. **预处理必要性**：

   - 当k较大时（如k=100），预处理能避免大量无效扫描
   - 小数据可省略，大数据必须有
