# 博弈类模拟问题通用解题模板

## 问题类型识别
适用于以下特征的博弈问题：
- 两个玩家轮流进行决策
- 每个回合的策略依赖于当前状态
- 游戏在特定条件下终止
- 需要模拟完整游戏过程

## 通用解题框架

### 1. 状态定义与初始化
```cpp
// 定义游戏状态变量
int player_turn = 0;           // 记录当前玩家 (0或1)
vector<ll> state(n);           // 存储各玩家的状态值
bool game_over = false;        // 游戏结束标志
```

### 2. 核心模拟循环
```cpp
while(!game_over)
{
    if(player_turn == 0)       // 玩家0的回合
    {
        state[0] = update_state(state[0], state[1], ...);
    }
    else                       // 玩家1的回合
    {
        state[1] = update_state(state[1], state[0], ...);
    }
    
    // 检查游戏结束条件
    game_over = check_game_end(state);
    
    // 切换玩家
    player_turn = 1 - player_turn;
}
```

### 3. 策略函数设计
```cpp
ll update_state(ll current, ll opponent, ...)
{
    if(condition_based_on(current, opponent))
    {
        return action1(current);  // 状态转移1
    }
    else
    {
        return action2(current);  // 状态转移2
    }
}
```

## 优化技巧

### 快速收敛检测
- **指数级操作**：如除法、开方等 → 循环次数很少 (log级别)
- **加法操作**：通常不会导致无限循环
- **周期性检查**：记录历史状态，避免死循环

### 边界条件处理
```cpp
// 通用结束条件模板
bool check_game_end(vector<ll>& state)
{
    // 条件1：某玩家资源耗尽
    for(int i = 0; i < n; i++)
    {
        if(state[i] == target_value) 
            return true;
    }
    
    // 条件2：达到最大轮次限制
    if(turn_count > max_turns) 
        return true;
        
    return false;
}
```

## 代码实现模板

```cpp
#include<bits/stdc++.h>
#define endl "\n"
using ll = long long;
using namespace std;

// 策略函数：根据当前状态决定行动
ll decide_action(ll self_state, ll other_state)
{
    // 根据具体题目条件填写策略逻辑
    if(self_state < other_state)
    {
        return self_state + increment;  // 获取资源
    }
    else
    {
        return self_state / divisor;    // 消耗资源
    }
}

int main()
{
    ll t;
    cin >> t;
    while(t--)
    {
        // 1. 初始化游戏状态
        vector<ll> players_state = {initial_val1, initial_val2, ...};
        int current_player = 0;  // 玩家编号
        
        // 2. 模拟游戏过程
        while(!game_ended(players_state))
        {
            // 执行当前玩家的策略
            players_state[current_player] = decide_action(
                players_state[current_player],
                players_state[1 - current_player]  // 对手状态
            );
            
            // 检查是否结束
            if(game_ended(players_state)) break;
            
            // 轮到下一位玩家
            current_player = 1 - current_player;
        }
        
        // 3. 输出结果
        output_result(players_state);
    }
    return 0;
}
```

## 常见变体及应对策略

| 变体类型 | 特征 | 应对方法 |
|---------|------|----------|
| 多玩家 | >2个玩家 | 使用模运算 `player_id % num_players` |
| 复杂状态 | 多维状态 | 使用结构体封装状态 |
| 动态规则 | 规则随轮次变化 | 将轮次作为策略函数参数 |
| 最优策略 | 需要博弈论最优解 | 考虑DP或SG函数 |

## 时间复杂度分析
- **收敛型**：O(log(max_state)) - 如本题的除法操作
- **线性型**：O(sum_of_states) - 如逐步减少资源
- **指数型**：O(2^n) - 需要考虑记忆化搜索

## Debug要点
1. **回合切换**：确保每次操作后正确切换玩家
2. **状态更新**：检查是否有变量未被更新
3. **边界条件**：特别注意初始状态和终止状态
4. **输入格式**：仔细核对题目要求的输入输出格式
